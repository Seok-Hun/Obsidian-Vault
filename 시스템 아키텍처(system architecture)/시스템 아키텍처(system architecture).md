# 정의
### 시스템의 구조, 작동을 위한 행위, 관련되는 뷰를 정의하는 <u>개념적 모형</u>

>[!개념적 모형]
>실질적인 프로그램들 간의 관계를 나타내는 High Level 수준의 조직 고유 모델
### 하드웨어와 소프트웨어를 모두 포함한 시스템의 전체적인 구성을 큰 그림으로 표현한 청사진
### 예시
1번의 전통적인 시스템의 단점 개선을 위해 고도화된 2번의 시스템 아키텍처 설계 및 구현
![[Pasted image 20231015003528.png]]

# 특징
### 설계된 아키텍처를 통해 기본적으로 시스템이 <font color="#00b0f0">어떤 하드웨어 기반</font>으로 소프트웨어를 사용하는지 알 수 있음
### 하드웨어/소프트웨어의 <font color="#00b0f0">시스템 간의 상호작용</font>이 정상작동을 위해 어떻게 동작하는지 쉽게 알 수 있음
### 기획, 분석, 설계, 구현, 테스트 및 운영을 위해 개발과 관련된 <font color="#00b0f0">이해관계자들의 이해</font>를 도울 수 있음
- 이해관계자 : PM, 개발자, 품질 및 테스트 관리자, 사용자 등
>[!PM(Project Manager)]
>한시적인 일을 수행하는 데 있어 관리 방법론에 따라 가장 효율적으로 추진하는 프로젝트의 계획과 실행에 있어서 종합적인 책임을 가진 직책 또는 책무
### 너무 복잡하면 안되고 <font color="#00b0f0">심플함</font>을 유지해야 한다.
